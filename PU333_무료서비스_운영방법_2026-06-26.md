# PHYSICAL UP 333 — 무료 서비스 운영 방법

> **작성일: 2026-06-26**
> 안티그래비티 개발팀 공유용 / 333테스트 무료 운영 설계

---

## 0. 핵심 전략 한 줄

> **무료를 "유료급 퀄리티"로 풀어서 충격 → 입소문 → 유행시킨다.**
> 원가가 0에 가까우니, 무료로 마음껏 뿌려도 손해가 없다. 모인 사람·데이터로 다른 데서 번다.

- 유료는 **밀지 않는다** (애 키로 장사 = 욕먹을 위험 + 돈도 애매).
- 유료는 "원하는 사람만" 조용히 둔다.
- **메인 수익은 쇼핑 추천**, 유료 아님.

---

## 1. 왜 무료인가

- **1순위 = 회원수 + 데이터.** 유료로 막으면 사람이 안 온다.
- 무료여야 사람이 오고, 와야 데이터가 쌓이고, 쌓여야 정부 지원·쇼핑 별도계약·구독이 강해진다.
- 보통은 "무료 = 대충, 유료 = 고급"인데, **우리는 반대로 무료부터 충격적 퀄리티.**
  → "공짜로 이런 걸?" → 카톡 공유 → 신뢰 폭발 → 유행.

---

## 2. 무료 vs 유료 구분

| 구분 | 입력 | 출력 |
|------|------|------|
| **무료** | 설문 + 키·몸무게·생년월일시 (BMI까지). **인바디 없음** | BioCode + 캐릭터 + 럭비공 분포 + 방향 + 제품 추천 |
| **유료** | + 인바디 + 부모키 + 목표 + 추가문항 | 정밀 분석 + 상세 운동 스케줄 + 로드맵 |

> **중요:** 코드는 무료·유료가 **같아야 함.**
> 무료는 인바디 자리를 키·몸무게·BMI로 대체(비중 고정) → **코드 안 흔들림** (검증 완료).
> 즉 무료도 지금 유료 보고서와 똑같이 멋진 PDF가 나온다 (인바디만 빠짐).

---

## 3. 운영 모델 — "제한으로 가치 만들기"

핵심 철학: **막을수록 더 갖고 싶고 더 자주 오게 된다.**

### ① 온라인이면서 예약제
- 온라인의 편함 + 오프라인의 격(예약해서 받는 느낌).
- 편한데 싸구려 느낌은 안 난다.

### ② 하루 50명 제한
- 희소성 = 마케팅. "오늘 마감" → 내일 더 빨리 → 입소문.
- 검수 감당되는 양 → 품질 유지 ("건전하게").
- 무료 한도(제미나이 일 약 700) 한참 안쪽 → **API 비용 ≈ 0.**

### ③ 한 달 전 예약 오픈
- "다음 달 예약 ○일 오픈!" → 티켓팅 효과, 그 자체가 이슈.
- 기다리는 한 달 = 부모가 준비하고 기대하는 시간 = 애착.

### ④ 정해진 날 오전 10시 일괄 발송
- 검수 시간 충분히 확보 (미리 받아두고 발송일까지 검수).
- 정해진 시간 동시 발송 → 그날 SNS 인증 쫙 → 또 마케팅.
- "매달 ○일 10시 발송" = 설렘의 정점, 하나의 의식(ritual).

### ⑤ 같은 사람 월 1회 제한
- 제한인데 사실은 **정기 구독 유도.** 아이는 매달 자라니 매달 체크가 당연해짐.
- **그래프도 월 1회 점.** 의미 있는 간격(매일 재면 오차만 출렁) → 깔끔한 성장 곡선.
- 첫 방문자는 점 하나 → "다음 달 오면 선이 그려져요" (다음을 기대).

### ⑥ 신중한 설문
- 빨리빨리 대충 X. 제대로 묻고 제대로 분석 → 신중함이 곧 신뢰 ("건전하게").

## 4. 단골화 + 서버 여유 (자동으로 맞물림)

- 월 1회씩 매달 오는 사람 = **단골·고정팬.** 입소문·후기·소개의 원천.
- 사람마다 방문일이 고정(누구는 1일, 누구는 15일) → **방문일 분산** → 하루에 안 몰림 → **서버·API 여유.**
- 단골 = 신뢰 = 추천 제품도 믿고 삼 → **정기 쇼핑 매출.**

> 1,500명(하루 50 × 30일) 단골이면 회사가 굴러간다.

---

## 5. 운영 사이클 (매달 반복되는 성장 엔진)

```
예약 오픈 (이슈)
   ↓
한 달 대기 (기대감)
   ↓
데이터 수집 (신중한 설문)
   ↓
검수 (품질 — 대표 검수)
   ↓
정해진 날 10시 일괄 발송 (설렘의 정점)
   ↓
SNS 인증 · 입소문
   ↓
다음 달 예약으로 (반복)
```

매 바퀴마다 **회원·데이터 누적 + 입소문 확대.**

---

## 6. 쇼핑 연결 (메인 수익 동선)

```
무료 테스트 → "이 아이는 흡수가 약함(대사1)" 진단
   → "그래서 이런 영양·식품·용품이 맞다" 추천 (엔진 근거 기반)
   → 쿠팡·네이버 링크 → 수수료
```

- 보고서 안에서 **체질·코드별 제품이 자동 추천**되게 연결.
- shop 페이지 = 구글시트 연동 → **링크만 바꾸면 수익 모드 전환** (파트너스 → 셀러 협상).
- 추천은 **데이터·체질 기준 먼저** → 신뢰 유지 (미션 안전장치).
